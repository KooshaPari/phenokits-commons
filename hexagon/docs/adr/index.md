## ADR Format

All ADRs in this repository follow a standard format based on the [nanovms/nanos](https://github.com/nanovms/nanos) decision record format:

1. **Title**: Descriptive title with ADR number
2. **Status**: Proposed, Accepted, Deprecated, Superseded
3. **Context**: Problem statement and decision drivers
4. **Decisions**: Specific decisions with rationale
5. **Consequences**: Positive and negative outcomes
6. **References**: Related documents and resources

## Contributing

When proposing a new ADR:

1. Use the next available ADR number
2. Follow the established format
3. Include traceability markers
4. Request review from architecture team
5. Update this index
